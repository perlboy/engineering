CDS Code Generator (cds-codegen)¶

Note

Development of this documentation is in progress.

Summary

The CDS Code Generator Suite is a multiple format code generation application centered around the Consumer Data Standards. It uses the programmatic model definition provided by CDS Models (cds-models) to output various versions of the CDS artefacts. Artefacts are intended to be regenerated using CDS Code Generator (cds-codegen) on a regular basis resulting in a unified perspective of the CDS Engineering artefacts. A significant portion of this documentation is generated using CDS Code Generator (cds-codegen).

Quick Start¶

Requirements¶

If you wish to simply run the CDS Code Generator you only require Docker.

If you wish to build CDS Codegen this requires:

  1. Java 1.8+

  2. Maven

Execution with Docker¶

To start the CDS Codegen execute the Docker command as below:

user@cds:~$ docker run -it consumerdatastandardsaustralia/cds-codegen-core:0.0.3-SNAPSHOT -h
Usage: <main class> [options]
  Options:
    --generator, -g
      Class name of cds-codegen generator
      Default: au.org.consumerdatastandards.codegen.generator.openapi.SwaggerGenerator
    --included, -i
      Include Section (comma separated)
    --excluded, -e
      Exclude Section (comma separated)
    --help, -?, -h

    --output-file, -o
      Output file to place generated swagger

Manual Build & Run¶

To manually build and run the CDS Code Generator using Maven, run:

user@cds:~/git$ git clone https://github.com/ConsumerDataStandardsAustralia/cds-codegen
user@cds:~/git$ cd cds-codegen/modules/cds-codegen-core
user@cds:~/git/cds-codegen/modules/cds-codegen-core$ mvn clean package
user@cds:~/git/cds-codegen/modules/cds-codegen-core$ java -jar target/cds-codegen-core-0.0.3-SNAPSHOT.jar  -h
